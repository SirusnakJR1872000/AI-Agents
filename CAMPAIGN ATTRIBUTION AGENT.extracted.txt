Data Ingestion Pathways
A. Salesforce Marketing Cloud (SFMC) Workflow To ensure a seamless flow of information from Salesforce, the architecture bifurcates data based on its point of origin:
Transactional Records: Purchase and sales data will flow from the Data cloud into SFMC (free of cost). From there, it will be pushed to a SFMC SFTP for external processing.
Campaign Engagement Metrics: Metadata related to campaign performance (opens, clicks, conversions) will be pulled directly from SFMC Data Extensions. This transfer will be managed via Automation Studio, which will automate the delivery of these extensions to the SFTP on a recurring schedule.
B. Netcore & Cloud Storage Workflow The integration for Netcore-supported brands will mirror the Salesforce logic but will utilize cloud-native storage for data landing:
Standardized Ingestion: Data from Netcore will be routed to brand-specific storage environments.
Infrastructure Variations: For brands such as Adidas, data will be pushed to dedicated Amazon S3 buckets. For entities like M&S, the system will adapt to their specific cloud infrastructure to ensure consistent data availability for the Python consolidation script.
For Netcore CDP – we recommend having campaign attribution setup as a feature directly within CDP
The Attribution Logic (Automation)
Identity Stitching & Ingestion: The system collects engagement logs (Email, SMS, etc.) and transaction data. It uses an Identity Resolution Layer to map Email/Phone/Customer IDs to create a Unified Customer View.
Event Sequencing: The engine checks that the campaign engagement (Click/Interaction) happened before the transaction date.
Attribution Window Filter: The system applies a "lookback" filter (e.g., 7 days). A transaction is only attributed if it occurs within X days of the campaign click.
Product & Category Mapping: The engine determines if the sale was Direct (the specific product targeted) or Indirect (a different brand or category under the same business unit).
